# 사용자 흐름

## 1. 타임라인/터미널 전환 흐름

```
1. 타임라인 탭 (기본) 표시 중
2. 네비게이션 바 우측 ⌨ (Terminal) 아이콘 클릭
3. 타임라인 fade out → 터미널 fade in (100ms)
4. 터미널 전체 화면 (축소 없이 100%)
5. xterm.js focus → 키보드 입력 가능
6. 💬 (MessageSquare) 아이콘 클릭 → 타임라인으로 복귀
```

## 2. 모바일 입력 흐름

```
1. 타임라인 탭에서 하단 입력창 터치
2. 모바일 키보드 올라옴
3. 입력창이 키보드 위에 자연스럽게 배치
4. 타임라인 영역이 위로 밀림 (허용)
5. 텍스트 입력 → Enter
6. 텍스트 + \r PTY 전송 (데스크톱과 동일)
7. 입력창 클리어, 포커스 유지
8. 키보드 유지 (연속 입력)
```

## 3. 메타 바 상세 확인 흐름

```
1. 컴팩트 메타 바 클릭
2. 바텀 시트 올라옴 (slide up, 200ms)
3. 상세 정보 표시 (시간, 메시지 수, 브랜치, 토큰)
4. 닫기:
   a. 드래그 핸들 아래로 스와이프
   b. 외부 (오버레이) 클릭
   c. 바텀 시트 내 아무 동작 없이 자동 닫힘 안 함
```

## 4. 세션 목록 → resume 흐름

```
1. Claude Code Panel에서 활성 세션 없음
2. 세션 목록 전체 화면 표시
3. 세션 항목 클릭
4. resume 진행 (데스크톱과 동일한 WebSocket 로직)
5. 성공: 타임라인 뷰로 전환
6. 차단/에러: sonner 토스트 표시
```

## 5. claude 명령어 실행 감지 흐름

```
1. 터미널 탭에서 사용자가 `claude` 입력
2. Phase 8 자동 감지 로직 동작
3. panelType → 'claude-code' 전환
4. 네비게이션 바에 터미널 토글 버튼 표시
5. 자동으로 타임라인 탭으로 전환
6. 타임라인 WebSocket subscribe
```

## 6. 엣지 케이스

### 터미널 탭에서 입력창과 xterm.js 포커스 충돌

```
터미널 탭: xterm.js가 전체 화면
├── xterm.js에 직접 터치 → xterm.js 포커스 (키보드 올라옴)
├── 입력창 터치 → 입력창 포커스 (xterm.js blur)
└── 둘 다 키보드를 올리므로 충돌 없음
    입력창은 Claude Code 메시지용, xterm.js는 직접 터미널 입력용
```

### 모바일에서 세션 종료 (inactive)

```
Claude Code 세션 종료 감지
├── 입력창 → 비활성 모드
├── 타임라인: 읽기 전용 유지
├── "← 세션 목록" 버튼으로 세션 목록 복귀 가능
└── 터미널 탭에서는 일반 터미널로 사용 가능
```

### 키보드가 화면 대부분을 차지

```
모바일 키보드가 화면의 40~50% 차지
├── 타임라인: 위로 밀려남 (일부만 보임) — 허용
├── 입력창: 키보드 바로 위에 위치
├── 메타 바: 보이지 않을 수 있음 — 허용
└── 키보드 닫히면 원래 레이아웃으로 복원
```
