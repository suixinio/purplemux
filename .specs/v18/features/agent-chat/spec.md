# 에이전트 채팅

## 개요

에이전트와 1:1로 대화하는 메신저 UI. 인턴에게 지시를 내리고, 질문에 답하고, 보고를 받는 느낌. 에이전트가 먼저 말을 걸 수 있으며, 메시지 유형별 시각적 구분으로 중요도를 직관적으로 파악한다.

## 주요 기능

### 메시지 표시

- 말풍선 기반 채팅 레이아웃 — 사용자(우측), 에이전트(좌측)
- 메시지 유형별 시각적 구분:
  - `report`: 기본 스타일 — 중간 보고, 진행 상황
  - `question`: 강조 배경 + 아이콘 — 사용자 응답 필요
  - `done`: 완료 뱃지 — 미션/태스크 완료 보고
  - `error`: 에러 스타일 — 실패 보고
  - `approval`: 승인 요청 스타일 + 승인/거부 액션 버튼
- 타임스탬프 표시 (날짜 구분선, 상대 시간)
- 새 메시지 도착 시 자동 스크롤 (사용자가 위로 스크롤 중이면 "새 메시지" 플로팅 버튼)

### 메시지 입력

- 하단 고정 입력 영역
- 텍스트 입력 + Enter 전송 (Shift+Enter 줄바꿈)
- 에이전트가 working 중이면 입력 비활성화 + "응답 대기 중..." 표시
- 에이전트가 offline이면 입력 비활성화 + "에이전트 오프라인" 표시

### 실시간 통신

- WebSocket으로 에이전트 메시지 수신 — 폴링 없음
- 에이전트가 working 중일 때 타이핑 인디케이터 표시 (점 3개 애니메이션)
- 연결 끊김 시 재연결 시도 + 상태 표시

### 에이전트 상태 헤더

- 채팅 상단에 에이전트 정보 표시: 이름, 역할, 현재 상태
- 상태 뱃지 실시간 갱신
- 에이전트 설정으로 이동하는 링크

### 채팅 이력

- 페이지 진입 시 최근 메시지 로드 (JSONL에서 역순 읽기)
- 위로 스크롤 시 이전 메시지 lazy load (무한 스크롤)
- 세션 구분: 날짜/시간 기반으로 대화 세션 구분선 표시

### 로딩/빈/에러 상태

- 초기 로딩: 메시지 스켈레톤
- 빈 상태 (첫 대화): "에이전트에게 첫 지시를 내려보세요" 안내
- WebSocket 연결 에러: 상단 배너로 연결 상태 표시
- 메시지 전송 실패: 해당 메시지에 재전송 버튼
