# 미션/태스크 현황

## 개요

에이전트가 수행 중인 Mission → Task → Step 3단계 트리를 시각화하고, 각 항목의 상태를 실시간으로 추적하는 대시보드. blocked 태스크를 즉시 식별하고 채팅으로 연결하여 빠르게 대응할 수 있다.

## 주요 기능

### 미션 목록

- 에이전트별 미션 리스트 (최신순)
- 미션 카드: 제목, 생성일, 전체 진행률 (완료 Task / 전체 Task), 상태 요약
- 활성 미션은 상단 고정, 완료된 미션은 접히는 아코디언
- 미션 상태 뱃지: running(진행 중 Task 있음) / blocked(blocked Task 있음) / completed(전체 완료) / failed

### 태스크 트리 뷰

- 미션 클릭 시 하위 Task → Step 트리 펼침
- 각 노드에 상태 아이콘:
  - `pending`: 회색 원
  - `running`: 녹색 펄스 원
  - `completed`: 체크마크
  - `blocked`: 주황 경고 아이콘 + 사유 표시
  - `failed`: 빨간 X + 에러 메시지
- 트리 노드 들여쓰기로 계층 표현
- 현재 실행 중인 Task/Step 강조 표시

### 실시간 상태 갱신

- WebSocket으로 상태 변경 수신 — 트리 노드 즉시 갱신
- 상태 전이 시 부드러운 애니메이션 (아이콘 전환, 색상 변경)
- 진행률 바 실시간 업데이트

### blocked 태스크 대응

- blocked 상태 노드 클릭 시 사유(에이전트의 질문) 팝오버 표시
- "채팅에서 답변하기" 버튼 → 채팅 페이지로 이동 (해당 질문 메시지로 스크롤)
- blocked 카운트 미션 카드에 뱃지 표시

### 병렬 태스크 시각화

- 동시 실행 중인 Task를 나란히 표시 (running 상태가 여러 개)
- 각 Task가 어느 워크스페이스/탭에서 실행 중인지 링크 표시
- 탭 클릭 시 해당 터미널 탭으로 이동하여 실시간 관찰 가능

### 롤링 계획 표시

- 아직 구체화되지 않은 Task는 점선 스타일로 표시 (확정 전)
- Task 시작 시 Step이 구체화되면 실선으로 전환
- "계획 조정됨" 표시 — 앞 단계 결과로 뒷 단계가 변경된 경우

### 로딩/빈/에러 상태

- 로딩: 트리 스켈레톤
- 빈 상태: "아직 미션이 없습니다" + 채팅에서 지시 유도
- WebSocket 연결 에러: 상단 배너
