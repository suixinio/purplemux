# 에이전트 목록/관리

## 개요

에이전트를 생성, 설정, 삭제하고 현재 상태를 한눈에 파악하는 관리 페이지. 헤더에서 진입하며, 각 에이전트의 실시간 상태(idle/working/blocked)를 표시한다.

## 주요 기능

### 에이전트 목록

- 카드 그리드 레이아웃 — 에이전트별 이름, 역할, 상태, 담당 프로젝트 표시
- 실시간 상태 뱃지: `idle`(회색) / `working`(녹색 펄스) / `blocked`(주황 펄스) / `offline`(비활성)
- 상태 변경 시 뱃지 즉시 전환 — WebSocket push 기반, 폴링 없음
- 빈 상태: 첫 에이전트 생성 유도 CTA

### 에이전트 생성

- 생성 다이얼로그 (Sheet 또는 Dialog)
  - 이름 (필수, 중복 불가)
  - 역할 (자유 텍스트 — "백엔드 개발", "코드 리뷰어" 등)
  - 담당 프로젝트 (기존 워크스페이스 목록에서 다중 선택)
- 생성 즉시 tmux 세션 시작 + config.md 저장
- 생성 완료 후 해당 에이전트 채팅 페이지로 이동

### 에이전트 설정 편집

- 카드 클릭 또는 설정 아이콘으로 설정 시트 열기
- 이름, 역할, 담당 프로젝트 수정
- 변경 즉시 config.md 반영

### 에이전트 삭제

- 삭제 확인 다이얼로그 (에이전트명 입력 확인)
- 삭제 시 tmux 세션 kill + chat 이력 + config 파일 정리
- 삭제 후 목록으로 복귀

### 헤더 진입점

- 앱 헤더에 에이전트 아이콘/버튼 추가
- blocked 에이전트가 있으면 뱃지로 알림 카운트 표시
- 클릭 시 `/agents` 페이지로 이동

### 로딩/에러 상태

- 목록 로딩: 스켈레톤 카드 표시
- API 에러: 토스트 알림 + 재시도 버튼
- 에이전트 생성/삭제 중: 버튼 비활성화 + 스피너
