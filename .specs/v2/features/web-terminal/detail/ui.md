# 화면 구성

> Phase 2에서 웹 터미널의 UI 변경은 최소한이다. 기존 전체 화면 터미널 레이아웃을 유지하며, 세션 종료 관련 UI만 보강한다.

## 전체 레이아웃 (변경 없음)

```
┌─────────────────────────────────────────────────────┐
│                                              [상태] │  ← 연결 상태 오버레이 (우상단, absolute)
│                                                     │
│                                                     │
│                   xterm.js 터미널                     │  ← 100vw × 100vh, 풀스크린
│                   (전체 화면)                         │
│                                                     │
│                                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 컴포넌트 구조 (변경 없음)

```
pages/index.tsx
└── <TerminalPage />
    ├── <TerminalContainer />          ← 터미널 영역 (풀스크린 div)
    │   └── xterm.js Terminal 인스턴스  ← ref로 마운트
    ├── <ConnectionStatus />           ← 연결 상태 오버레이
    └── <SessionEndedOverlay />        ← 세션 종료 오버레이
```

## 상태별 UI (Phase 2 변경 반영)

| 상태 | UI | Phase 1 대비 변경 |
|---|---|---|
| `connected` | 표시 없음 (정상 상태는 비표시) | 동일 |
| `connecting` | 중앙 스피너 + "연결 중..." | 동일 |
| `reconnecting` | 재연결 시도 중 + 횟수 | 동일 (재연결 후 화면 복원됨) |
| `disconnected` | 연결 끊김 알림 + 재연결 버튼 | 동일 |
| `session-ended` | 하단 오버레이 + "새 세션 시작" 버튼 | 동일 |

### reconnecting → connected 전환 시 UX 변화

Phase 1에서는 재연결 성공 후 빈 터미널(새 PTY)이 표시되었지만, Phase 2에서는:

- 재연결 성공 → tmux가 화면 redraw → 이전 터미널 상태가 복원됨
- reconnecting 상태 오버레이가 fade out (150ms)되면서 이전 화면이 자연스럽게 나타남
- 사용자 체감: "잠깐 끊겼다가 원래대로 돌아옴"

### 새로고침 시 UX

```
새로고침 → 페이지 리로드
→ connecting 상태 (중앙 스피너)
→ WebSocket 연결 성공
→ tmux attach → 화면 redraw
→ connected 상태 (스피너 fade out)
→ 이전 터미널 상태 표시
```

- xterm.js 인스턴스는 페이지 리로드로 새로 생성되지만, tmux redraw로 이전 화면이 복원됨
- connecting → connected 전환이 500ms 이내로 빠르게 이루어져야 함

## 세션 종료 오버레이 (SessionEndedOverlay)

Phase 1과 동일한 구조. 변경 없음:

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│                   xterm.js 터미널                     │
│                   (마지막 화면 유지)                   │
│                                                     │
├─────────────────────────────────────────────────────┤
│  ⚠ 세션이 종료되었습니다.        [새 세션 시작]       │  ← 하단 오버레이
└─────────────────────────────────────────────────────┘
```

- close code 1000 수신 시 표시
- "새 세션 시작" 클릭 → 새 WebSocket 연결 → 서버가 새 tmux 세션 생성
