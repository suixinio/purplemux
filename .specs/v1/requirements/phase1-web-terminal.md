# Phase 1 — 웹 터미널 PRD

## 목표

브라우저에서 터미널 하나가 동작하는 것.

이후 Phase의 기반이 되는 최소 동작 단위를 구현한다. tmux, 탭, 분할 등 복잡한 기능 없이 순수하게 "웹에서 터미널을 쓸 수 있는가"만 검증한다.

## 완료 조건

`localhost:{port}`에 접속하면 터미널이 뜨고 명령어 실행이 가능하다.

---

## 요구사항

### REQ-1: Next.js 서버

Next.js 기본 서버를 사용한다. Custom Server 없이 API Route를 통해 WebSocket을 처리한다.

- `pnpm dev` / `pnpm start`로 서버를 구동한다
- 페이지 렌더링은 Next.js Pages Router가 처리한다
- WebSocket은 API Route(`/api/terminal`)에서 HTTP 연결을 업그레이드하여 처리한다

### REQ-2: PTY 프로세스 생성

클라이언트가 WebSocket으로 연결하면, 서버에서 가상 터미널(PTY)을 생성하여 쉘을 실행한다.

- `node-pty`를 사용하여 사용자의 기본 쉘을 실행한다
- PTY의 stdout을 WebSocket으로 클라이언트에 전송한다
- 클라이언트의 입력을 WebSocket으로 받아 PTY의 stdin에 전달한다
- 연결 종료 시 PTY 프로세스를 정리한다

### REQ-3: 웹 터미널 UI

브라우저에서 터미널을 렌더링하고 입력을 처리한다.

- `xterm.js`를 사용하여 터미널을 렌더링한다
- 터미널이 브라우저 화면 전체를 차지한다 (전체 화면 터미널)
- 키 입력을 WebSocket으로 서버에 전송한다
- 서버로부터 받은 출력을 터미널에 렌더링한다

### REQ-4: WebSocket 통신

클라이언트와 서버 간 양방향 실시간 통신을 제공한다.

- 페이지 로드 시 자동으로 WebSocket 연결을 수립한다
- 바이너리 데이터를 처리할 수 있어야 한다 (컬러 출력, 특수 문자 등)
- 연결이 끊어진 경우 사용자에게 알린다

### REQ-5: 터미널 리사이즈

브라우저 창 크기 변경 시 터미널 크기가 자동으로 조정된다.

- 브라우저 리사이즈 이벤트를 감지하여 터미널 크기를 재계산한다
- 변경된 크기(cols, rows)를 서버에 전달한다
- 서버에서 PTY의 크기를 업데이트한다

## 비기능 요구사항

### NFR-1: 지연 시간

키 입력에서 화면 출력까지 체감 지연이 없어야 한다. (로컬 환경 기준)

### NFR-2: 호환성

일반 터미널에서 동작하는 CLI 도구(vim, htop, git 등)가 정상 동작해야 한다.

### NFR-3: 한글 입력

한글 IME 입력이 정상적으로 처리되어야 한다.

## 기술 구성

```
Browser                          Server (Next.js)
┌──────────────┐    WebSocket    ┌──────────────────────┐
│  xterm.js    │ ◄────────────► │  /api/terminal       │
│  (터미널 UI)  │                │  (node-pty + ws)     │
│              │    HTTP         │                      │
│  Next.js     │ ◄────────────► │  Pages Router        │
│  (페이지)     │                │  (SSR/정적 서빙)      │
└──────────────┘                └──────────────────────┘
```

### 주요 라이브러리

| 용도 | 라이브러리 |
|---|---|
| 프레임워크 | Next.js (Pages Router) |
| 터미널 렌더링 | xterm.js |
| 가상 터미널 | node-pty |
| WebSocket | ws |
| 프론트엔드 | React |

## 검증 시나리오

1. **기본 명령어 실행**: `ls`, `echo`, `pwd` 등 기본 명령어가 정상 동작한다
2. **인터랙티브 프로그램**: `vim`, `htop` 등 TUI 프로그램이 정상 동작한다
3. **컬러 출력**: `ls --color`, git diff 등 ANSI 컬러가 올바르게 렌더링된다
4. **한글 입력/출력**: 한글 파일명 생성, 한글 텍스트 echo가 정상 동작한다
5. **리사이즈**: 브라우저 창 크기를 변경하면 터미널이 자연스럽게 재조정된다
6. **장시간 출력**: `find /` 같은 대량 출력이 멈추지 않고 처리된다
