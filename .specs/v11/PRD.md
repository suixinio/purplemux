| 페이지/뷰 | 설명 | 우선순위 |
|---|---|---|

## 주요 요구사항

### 메타 바 UI

- 타임라인 영역 최상단에 고정 배치 (스크롤 밖)
- 컴팩트 레이아웃 (기본): 한 줄에 제목 · 시간 · 턴 수 · 토큰 요약
- 상세 레이아웃 (클릭 확장): 생성/수정 시간, 사용자/어시스턴트 메시지 수, git 브랜치, 입력/출력 토큰 분리
- 세션 목록 뷰에서는 숨김, 타임라인 뷰에서만 표시

### 표시 항목

| 항목 | 데이터 소스 | 컴팩트 표시 | 상세 표시 |
|---|---|---|---|
| 세션 제목 | 첫 user-message 또는 summary 필드 | truncate 텍스트 | 전체 텍스트 |
| 시간 | 첫/마지막 엔트리 타임스탬프 | 상대 시간 (마지막) | 생성 + 수정 시간 |
| 메시지 수 | 엔트리 카운트 | 총 턴 수 | 사용자 N / 어시스턴트 N |
| git 브랜치 | tmux cwd → git HEAD | — (컴팩트에 생략) | 브랜치명 |
| 토큰 사용량 | usage 필드 집계 | 총합 (K 단위) | 입력/출력/총합 |

### 메타데이터 파싱

- 기존 세션 파서를 확장하여 usage 필드 추출
- 토큰: `input_tokens`, `output_tokens` 집계 — 증분 계산 (timeline:append 시)
- 제목: summary 필드 우선, 없으면 첫 user-message
- 메시지 수: 기존 타임라인 엔트리에서 파생 (추가 파싱 불필요)

### git 브랜치 조회

- tmux 세션 cwd에서 `git rev-parse --abbrev-ref HEAD` 실행
- 서버 API (`GET /api/git/branch?tmuxSession={name}`)로 제공
- 주기적 폴링 (30초 간격) — 브랜치 변경은 빈번하지 않으므로 폴링으로 충분

### 실시간 갱신

- 기존 `timeline:append` WebSocket 수신 시 메타 바 자동 갱신
- 토큰: 새 엔트리의 usage를 기존 합계에 더함 (O(1))
- 메시지 수: 엔트리 타입에 따라 카운터 증가
- 최종 수정 시간: 새 엔트리 타임스탬프로 갱신

## 제약 조건 / 참고 사항

### 기술적 제약

- **usage 필드 가용성**: Claude Code 세션 파일의 모든 메시지에 usage 필드가 있는지 확인 필요. 없는 메시지는 0으로 처리
- **git 명령어 실행 비용**: `git rev-parse`는 가벼운 명령이지만, 매 엔트리마다 호출하면 안 됨 → 30초 폴링
- **세션 파서 확장**: 기존 `session-parser.ts`의 파싱 스키마에 usage 필드를 추가해야 함

### UX 고려사항

- **토큰 단위**: 1000 이상은 K 단위 표시 (예: 50,234 → "50.2K"), 100만 이상은 M 단위
- **시간 표시**: dayjs `fromNow` + 호버 시 절대 시간 Tooltip
- **컴팩트 → 상세 전환**: 클릭으로 토글, 부드러운 height 애니메이션
- **정보 밀도**: 컴팩트 모드는 한눈에 파악 가능한 수준, 너무 많은 정보를 욱여넣지 않음

### 성능

- 메타 바 렌더링은 기존 데이터(타임라인 엔트리)에서 파생 — 추가 API 호출 없음 (git 브랜치 제외)
- 토큰 집계는 증분 계산 — 대용량 세션에서도 O(1) 업데이트
