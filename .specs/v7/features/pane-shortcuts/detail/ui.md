# 화면 구성

## 개요

UI 레이아웃 변경 없음. Pane 분할/포커스 이동 단축키는 기존 UI 버튼과 동일한 함수를 호출하며, 시각적 결과도 동일하다.

## Pane 분할 (⌘D / ⌘⇧D)

### 분할 전

```
┌─────────────────────────────────────┐
│ [Tab1] [Tab2]        [─] [│] [✕]   │ ← 탭 바 (기존)
├─────────────────────────────────────┤
│                                     │
│            Terminal                  │
│                                     │
└─────────────────────────────────────┘
```

### ⌘D 수직 분할 후

```
┌──────────────────┐┃┌──────────────────┐
│ [Tab1] [Tab2]    │┃│ [Tab3]  [─][│][✕]│ ← 포커스 이동
├──────────────────┤┃├──────────────────┤
│                  │┃│                  │
│   Terminal (기존) │┃│  Terminal (신규)  │
│                  │┃│                  │
└──────────────────┘┃└──────────────────┘
                    ↑ 리사이즈 핸들
```

### ⌘⇧D 수평 분할 후

```
┌─────────────────────────────────────┐
│ [Tab1] [Tab2]          [─] [│] [✕]  │
├─────────────────────────────────────┤
│            Terminal (기존)            │
│                                     │
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ← 리사이즈 핸들
┌─────────────────────────────────────┐
│ [Tab3]                 [─] [│] [✕]  │ ← 포커스 이동
├─────────────────────────────────────┤
│            Terminal (신규)            │
└─────────────────────────────────────┘
```

### 시각적 차이

없음. 탭 바 분할 버튼 클릭과 완전히 동일한 결과.

### 분할 불가 상태

- `canSplit === false` (paneCount >= 10 또는 isSplitting === true) 시 단축키 무시
- 별도 에러 표시 없음 (버튼 클릭 시에도 동일)

## Pane 포커스 이동 (⌥⌘ + 방향키)

### 포커스 표시

기존 포커스 Pane 하이라이트 스타일을 그대로 활용. 탭 바의 시각적 구분으로 포커스 상태를 표시.

### 이동 예시: 수직 2분할에서 ⌥⌘→

```
BEFORE:                                AFTER:
┌──────────┐┃┌──────────┐           ┌──────────┐┃┌──────────┐
│ ★ Pane A │┃│   Pane B │           │   Pane A │┃│ ★ Pane B │
│ (focused)│┃│          │    →      │          │┃│ (focused)│
└──────────┘┃└──────────┘           └──────────┘┃└──────────┘

★ = 포커스 표시 (탭 바 하이라이트)
```

### 이동 예시: 복합 분할

```
┌──────────┐┃┌──────────┐
│          │┃│  Pane B  │
│ ★ Pane A │┃├──────────┤     ⌥⌘→ → Pane B로 이동
│          │┃│  Pane C  │     ⌥⌘↓ → 이동 불가 (하단에 Pane 없음)
└──────────┘┃└──────────┘
```

### 방향에 Pane 없을 때

아무 시각적 변화 없음. 피드백 없이 무시.

## 상태별 화면

### 로딩 상태 (layout === null)

- 단축키 입력 무시
- 기존 로딩 UI 그대로 표시 (탭 바 스켈레톤 + 중앙 스피너)

### 분할 진행 중 (isSplitting)

- 추가 분할 단축키 무시
- 기존 분할 버튼 비활성 상태와 동일
