# 화면 구성

## 개요

UI 레이아웃 변경 없음. 탭 단축키는 기존 탭 바 UI(+버튼, X버튼, 탭 클릭)와 동일한 함수를 호출하며, 시각적 결과도 동일하다.

## 새 탭 생성 (⌘T)

### 탭 추가 후

```
BEFORE:                              AFTER:
┌──────────────────────────┐         ┌──────────────────────────────┐
│ [Tab1] [★Tab2]    [+]   │         │ [Tab1] [Tab2] [★Tab3]  [+]  │
├──────────────────────────┤         ├──────────────────────────────┤
│      Terminal            │         │      Terminal (새 탭)         │
└──────────────────────────┘         └──────────────────────────────┘

★ = 활성 탭
```

기존 탭 바 `+` 버튼 클릭과 동일한 결과.

## 탭 닫기 (⌘W)

### 탭 2개 이상일 때

```
BEFORE:                              AFTER:
┌──────────────────────────┐         ┌──────────────────────────┐
│ [Tab1] [★Tab2] [Tab3]   │         │ [Tab1] [★Tab3]           │
├──────────────────────────┤         ├──────────────────────────┤
│      Terminal            │         │      Terminal            │
└──────────────────────────┘         └──────────────────────────┘
```

### 마지막 탭 닫기 → Pane 제거

```
BEFORE (2 Pane):                     AFTER (1 Pane):
┌────────┐┃┌────────┐               ┌─────────────────┐
│ [★Tab1]│┃│ [Tab2] │               │ [★Tab2]         │
│ Term   │┃│ Term   │    ⌘W →       │ Terminal         │
└────────┘┃└────────┘               └─────────────────┘
   ↑ 포커스                            ↑ 포커스 이동
```

### Workspace 최후의 Pane/탭

기존 X 버튼 동작과 동일하게 처리. 빈 Workspace 상태에 대한 별도 UI 없음.

## 이전/다음 탭 전환 (⌘⇧[ / ⌘⇧])

### 탭 전환

```
탭 순서: [Tab1] [★Tab2] [Tab3]

⌘⇧[ → [★Tab1] [Tab2] [Tab3]   (이전)
⌘⇧] → [Tab1] [Tab2] [★Tab3]   (다음)
```

### 끝에서 멈춤

```
탭 순서: [★Tab1] [Tab2] [Tab3]

⌘⇧[ → [★Tab1] [Tab2] [Tab3]   (변화 없음 — 첫 번째 탭)
```

## 번호로 탭 이동 (⌃1~9)

```
탭 순서: [Tab1] [★Tab2] [Tab3] [Tab4]

⌃1 → [★Tab1] [Tab2] [Tab3] [Tab4]   (1번째)
⌃4 → [Tab1] [Tab2] [Tab3] [★Tab4]   (4번째)
⌃9 → [Tab1] [Tab2] [Tab3] [★Tab4]   (마지막)
⌃5 → 변화 없음 (존재하지 않는 번호)
```

## 탭 전환 피드백

- 활성 탭 하이라이트 스타일 즉시 갱신 (기존 탭 클릭과 동일)
- 터미널 내용 즉시 전환 (WebSocket 연결 전환)
- 포커스 자동 이동 (xterm.js focus())
