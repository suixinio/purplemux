# 화면 구성

## 개요

UI 레이아웃 변경 없음. Workspace 전환 단축키는 기존 사이드바 클릭과 동일한 함수를 호출하며, 터미널 스크롤백 지우기는 xterm.js 내장 메서드를 사용한다.

## Workspace 전환 (⌘1~9)

### 전환 전후

```
BEFORE (Workspace 1 활성):

┌──────────┐┃┌────────────────────────────┐
│ ★ ws-1   │┃│ [Tab1] [Tab2]              │
│   ws-2   │┃│ Terminal (ws-1)             │
│   ws-3   │┃│                            │
└──────────┘┃└────────────────────────────┘

⌘2 입력 →

AFTER (Workspace 2 활성):

┌──────────┐┃┌────────────────────────────┐
│   ws-1   │┃│ [Tab3]                     │
│ ★ ws-2   │┃│ Terminal (ws-2)             │
│   ws-3   │┃│                            │
└──────────┘┃└────────────────────────────┘

★ = 활성 Workspace 하이라이트
```

### 전환 애니메이션

기존 사이드바 클릭 전환과 동일:

```
fadeOut 100ms → clearLayout → fetch 새 layout → fadeIn 100ms
```

### 존재하지 않는 번호

Workspace 2개인데 `⌘5` → 아무 시각적 변화 없음.

### 이미 활성 Workspace

현재 활성 Workspace의 번호 입력 → 무시 (불필요한 전환 방지).

## 터미널 스크롤백 지우기 (⌘K)

### 지우기 전

```
┌─────────────────────────┐
│ $ ls                    │  ↑
│ file1.txt  file2.txt    │  │ 스크롤백 영역
│ $ npm install           │  │ (히스토리)
│ added 100 packages      │  │
│ $ echo hello            │  ↓
│ hello                   │
│ $                       │ ← 현재 커서
└─────────────────────────┘
```

### 지우기 후

```
┌─────────────────────────┐
│                         │
│                         │
│                         │
│                         │
│ $                       │ ← 현재 커서 (유지)
└─────────────────────────┘
```

- 현재 화면의 프롬프트는 유지
- 스크롤백 히스토리만 제거
- 스크롤 위치는 최하단으로 리셋
